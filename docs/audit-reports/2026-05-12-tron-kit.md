**AC5 §4 exception — `try_optional_swallow` ObjectMapper FP pattern (CTO approved 2026-05-13)**

The `error_handling_policy` extractor's `try_optional_swallow` rule flags 5/5 of the top-5
§4 findings as false positives. These are ObjectMapper `TransformType`/`Mappable` idioms:

- `Transforms.swift:63,77` — `HexAddressTransform` / `StringAddressTransform` return `Address?`
  via `try? Address(raw:)` / `try? Address(address:)`. Idiomatic nil-on-invalid-input.
- `Contract.swift:157,438,439` — `try? map.value(...)` for optional JSON fields in `Mappable`
  init. Standard ObjectMapper pattern for optional contract parameters.

Real TPs exist in §4 (`catch_only_logs` in `SendController.swift:168` and
`Trc20SendController.swift:177`) but appear outside the top-5.

**Root cause:** extractor lacks context-aware suppression for framework-idiomatic `try?`
patterns (ObjectMapper `TransformType` conformance, `Mappable.init(map:)` bodies).

**S4.4 follow-up:** add FP suppression rule — skip `try_optional_swallow` when enclosing
method matches `TransformType.transformFromJSON` signature or `Mappable.init(map:)` body.
