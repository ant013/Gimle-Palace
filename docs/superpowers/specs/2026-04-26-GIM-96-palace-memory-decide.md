# GIM-96 — `palace.memory.decide` — write-side `:Decision` MCP tool

## Goal

Close the **read-write loop** for `:Decision` nodes in Graphiti. Today, agents read decisions via `palace.memory.lookup(entity_type="Decision", ...)`, but no agent (or operator) can RECORD a new `:Decision`. Result: Graphiti's product/process layer has bridge-extractor data (Symbol/File/Module — code-side) but **zero `:Decision` nodes** (process-side). The most valuable conceptual layer of N+1a is empty.

This slice ships `palace.memory.decide(...)` MCP tool that creates a `:Decision` `EntityNode` in Graphiti with the metadata envelope established by GIM-77 bridge extractor (provenance, confidence, decision_maker, slice_ref, etc.). After this lands, every CR APPROVE / Opus verdict / operator design call can be recorded by the agent that produced it.

## Sequence

Per operator decision 2026-04-26, this is **Slice 1 of 4** in N+2 Category 1 (USE-BUILT) — the ENABLER:

1. **`palace.memory.decide` — this slice (ENABLER)**
2. `palace.context.prime` — per-role priming (GIM-95 spec drafted)
3. `palace.code.test_impact(qn)` — composite tool
4. `palace.code.semantic_search` — hybrid retrieval

**Why first:** without `decide`, priming returns empty `:Decision` lists for every slice. With `decide` shipped, agents start recording decisions; priming starts showing them; the feedback loop closes.

**Hard dependencies:**
- N+1a Graphiti foundation (GIM-75) — ✅ landed (`cf2fd0f`)
- `graphiti_runtime.py` helpers `save_entity_node`, `generate_name_embedding` (post-QA fix `036104e`) — ✅ landed

## Non-goals

- Auto-recording from CR/Opus/QA APPROVE comments (out of scope; manual call by agent for v1)
- Decision supersession / amendment / revocation flow — separate slice if needed
- Cross-slice decision linking (e.g., "this decision overrides Decision X") — separate slice
- UI for browsing decisions — `palace.memory.lookup` is the read path; UI not in scope
- Versioning / mutation history — Graphiti has bi-temporal model (`valid_at`/`invalid_at`); we'll record `valid_at=now` and let supersession be a future slice

## Architecture

### Decision schema (`:Decision` `EntityNode`)

Per operator decision 2026-04-26 Q3: minimum useful fields. We extend later if needed.

**Required fields:**
- `name` (Pydantic `EntityNode.name`) — short title, used for embedding + lookup
- `group_id` — `project/gimle` (default; configurable per `Settings.palace_default_group_id`)
- `attributes` (dict, first-class in graphiti-core 0.28):
  - `body: str` — full decision text (Markdown allowed)
  - `slice_ref: str` — slice ID (e.g. `"GIM-96"`); required for filtering
  - `decision_maker: str` — agent role or operator (one of `cto, codereviewer, pythonengineer, opusarchitectreviewer, qaengineer, operator`)
  - `provenance: str` — fixed value `"asserted"` (per GIM-77 metadata convention)
  - `confidence: float` — `0.0..1.0`, default `1.0`
  - `decided_at: datetime ISO8601` — when decision was made; defaults to `now`
  - `extractor: str` — fixed value `"palace.memory.decide@0.1"`
  - `extractor_version: str` — `"0.1"`

**Optional fields (in `attributes`):**
- `tags: list[str]` — free-form keywords (e.g. `["security", "deferred"]`)
- `evidence_ref: list[str]` — external references (PR URLs, commit SHAs, paperclip issue UUIDs)
- `supersedes: list[str]` — UUIDs of `:Decision` nodes this one replaces (for future supersession; v1 just stores; doesn't enforce semantics)

**Auto-computed:**
- `uuid` — generated by Graphiti
- `created_at` — Graphiti default
- `name_embedding` — via `generate_name_embedding(g.embedder)` per GIM-75 QA fix
- `labels: ["Decision"]` — set explicitly in node creation

**Validation:**
- `title` must be 1..200 chars (suitable for embedding)
- `body` must be 1..10000 chars (rough cap — Graphiti can store more, but keep records focused)
- `slice_ref` must match regex `^GIM-\d+$|^N\+\d+\w*$|^operator-decision-\d{8}$` (for ad-hoc operator decisions)
- `decision_maker` must be one of the canonical agent role keys + `"operator"` + `"board"`
- `confidence` must be `0.0..1.0`

**Returns:**
```json
{
  "ok": true,
  "uuid": "<graphiti-generated-uuid>",
  "name": "<title>",
  "slice_ref": "GIM-96",
  "decision_maker": "operator",
  "decided_at": "2026-04-26T05:30:12Z",
  "name_embedding_dim": 1536
}
```

On error (validation failure, embedder failure, Neo4j failure):
```json
{
  "ok": false,
  "error_code": "validation_error|embedder_failure|neo4j_failure|unknown",
  "message": "<short explanation>"
}
```

### Task 1 — DecideRequest

```python
from datetime import datetime
from re import match as re_match
from pydantic import BaseModel, Field, field_validator

VALID_AGENT_ROLES = {
    "cto", "codereviewer", "pythonengineer",
    "opusarchitectreviewer", "qaengineer",
    "operator", "board",
}

class DecideRequest(BaseModel):
    title: str = Field(..., min_length=1, max_length=200)
    body: str = Field(..., min_length=1, max_length=10_000)
    slice_ref: str = Field(..., min_length=1, max_length=80)
    decision_maker: str = Field(..., min_length=1)
    tags: list[str] | None = None
    evidence_ref: list[str] | None = None
    confidence: float = Field(1.0, ge=0.0, le=1.0)
    supersedes: list[str] | None = None

    @field_validator("slice_ref")
    @classmethod
    def _slice_ref_format(cls, v: str) -> str:
        if not re_match(r"^GIM-\d+$|^N\+\d+\w*$|^operator-decision-\d{8}$", v):
            raise ValueError(f"slice_ref '{v}' must match GIM-N | N+Nx | operator-decision-YYYYMMDD")
        return v

    @field_validator("decision_maker")
    @classmethod
    def _decision_maker_allowed(cls, v: str) -> str:
        if v not in VALID_AGENT_ROLES:
            raise ValueError(f"decision_maker '{v}' not in {VALID_AGENT_ROLES}")
        return v
```

### Task 3 — MCP registration

In `mcp_server.py`, add via `_tool()` wrapper (Pattern #21):

```python
@_tool(
    name="palace.memory.decide",
    description="...",
)
async def palace_memory_decide(
    title: str,
    body: str,
    slice_ref: str,
    decision_maker: str,
    tags: list[str] | None = None,
    evidence_ref: list[str] | None = None,
    confidence: float = 1.0,
    supersedes: list[str] | None = None,
) -> dict[str, Any]:
    g = _graphiti
    if g is None:
        return {"ok": False, "error_code": "service_unavailable", "message": "graphiti not initialized"}
    try:
        req = DecideRequest(
            title=title, body=body, slice_ref=slice_ref,
            decision_maker=decision_maker, tags=tags,
            evidence_ref=evidence_ref, confidence=confidence,
            supersedes=supersedes,
        )
    except ValidationError as e:
        return {"ok": False, "error_code": "validation_error", "message": str(e)}
    return await decide(req, g=g, default_group_id=_default_group_id)
```

## Open questions for operator review

1. **`decision_maker` enum:** include `"board"` as an alias for `"operator"`, or merge them? Currently both allowed — minor redundancy. Operator preference?

2. **`slice_ref` format strictness:** regex allows `GIM-N | N+Nx | operator-decision-YYYYMMDD`. Add free-form fallback if useful? Or keep strict for filterability?

3. **`body` cap of 10000 chars:** is this too generous (encourages bloat in the prompt cache when read-back) or too tight (makes long technical decisions unfit)?

4. **Confidence semantic:** is `confidence` filled by the agent reflecting their own certainty, or by a CR/Opus reviewer's assessment? Currently unspecified semantically. Leave to convention or document?

5. **Multiple decisions per slice:** allowed (no uniqueness constraint), can be many. Lookup orders by `created_at desc`. Sufficient or need `decision_kind` discriminator (e.g., "design", "review-approve", "scope-change")?
