# Dead Symbol Tool Output Spike

**Date:** 2026-05-05
**Issue:** GIM-193
**Tool:** `Periphery`

## Goal

Зафиксировать реальный `Periphery` input contract для roadmap #33 до начала
parser implementation и отдельно зафиксировать, что `Reaper` не даёт публичный
offline report contract для v1.

## Environment

- Repo branch: `feature/roadmap-33-dead-symbol-binary-surface`
- Host OS: macOS 13
- Host Swift: `Apple Swift version 5.8.1`
- Host developer tools: Command Line Tools only, no full `Xcode.app`
- Homebrew formula metadata:
  - `periphery` stable: `3.7.4`
  - build requirement: `Xcode.app 16.4`

## Commands

### 1. Проверка локальной доступности `Periphery`

```bash
which periphery && periphery version || true
brew info periphery --json=v2
```

Observed:

- `periphery` is not installed on the host.
- Homebrew reports stable version `3.7.4`.

### 2. Проверка локальной Swift/Xcode среды

```bash
swift --version
xcodebuild -version || true
xcrun --sdk macosx --show-sdk-path
xcrun --sdk macosx --show-sdk-platform-path
```

Observed:

- `swift --version` -> `Apple Swift version 5.8.1 (swiftlang-5.8.0.124.5 clang-1403.0.22.11.100)`.
- `xcodebuild` fails because the active developer directory is Command Line Tools.
- `xcrun --show-sdk-path` succeeds with
  `/Library/Developer/CommandLineTools/SDKs/MacOSX.sdk`.
- `xcrun --show-sdk-platform-path` fails from CLT-only install, which also
  breaks local `swift build` / `swift package describe` for this fixture.

### 3. Попытка установить `Periphery` на host

```bash
brew install --formula --force-bottle periphery
```

Observed:

- Homebrew refuses installation: `periphery: A full installation of Xcode.app 16.4 is required to compile this software. Installing just the Command Line Tools is not sufficient.`
- Same error notes that `Xcode 16.4 cannot be installed on macOS 13`.

### 4. Официальные release assets `Periphery 3.7.4`

```bash
gh api repos/peripheryapp/periphery/releases/tags/3.7.4 --jq '.assets[] | [.name,.browser_download_url] | @tsv'
curl -L --fail -o /tmp/periphery-3.7.4.zip https://github.com/peripheryapp/periphery/releases/download/3.7.4/periphery-3.7.4.zip
unzip -q /tmp/periphery-3.7.4.zip -d /tmp/periphery-release
find /tmp/periphery-release -maxdepth 4 -type f | sort
DYLD_LIBRARY_PATH=/tmp/periphery-release /tmp/periphery-release/periphery version
```

Observed:

- Release assets exist:
  - `periphery-3.7.4.artifactbundle.zip`
  - `periphery-3.7.4.zip`
  - `periphery-bazel-3.7.4.zip`
- The plain release zip contains:
  - `periphery`
  - `libIndexStore.dylib`
  - `LICENSE.md`
- The artifact bundle still only contains a macOS binary payload.
- Both binaries fail on the current host because they are built for newer
  macOS and require `/usr/lib/swift/libswiftSynchronization.dylib`, which is
  not present on this macOS 13 runner.

- Periphery official README:
  <https://github.com/peripheryapp/periphery>
  documents:
  - SwiftPM support on macOS and Linux
  - `--retain-public` for framework/library-style projects

- Emerge Reaper docs:
  <https://docs.emergetools.com/docs/reaper>,
  <https://docs.emergetools.com/docs/reaper-setup>,
  <https://docs.emergetools.com/docs/reaper-setup-android>

  stable offline report-file contract for extractor ingestion.

## Observations

- Gate 0 cannot be completed purely on the current macOS host because the host
  lacks a full supported Xcode installation, and Homebrew confirms that the
  current `periphery` release requires `Xcode.app 16.4`.
- Official `Periphery 3.7.4` release assets do not unblock the host because the
  shipped macOS binary targets a newer OS/runtime than this runner provides.
- The blocker is environmental, not spec-related: the SwiftPM fixture is ready,
  but the tool cannot yet be run locally.
- Reaper remains v1 schema-ready/no-op only. Current docs continue to describe
  network upload and backend aggregation rather than a file export contract we
  could safely parse offline.
- A Linux/Docker fallback is being explored because Periphery documents SwiftPM
  support on Linux. If that fallback succeeds, this README will be updated with
  the exact raw JSON contract and fixture artifacts; if it fails, Gate 0 should
  stay blocked on environment/tool availability.

## Decision For Gate 0

- Do not start parser implementation until a real `Periphery` output capture is
  committed and signed off by `CXCodeReviewer`.
- Current evidence justifies an environment blocker on this runner.
- Unblock options:
  1. run Gate 0 on a supported macOS host with `Xcode.app 16.4+`; or
  2. provide/approve a Linux-distributed `Periphery` binary compatible with the
     SwiftPM fixture path so the raw JSON contract can be captured there.
